Say, there is a researcher who wants to use a cloud computing service to run a machine learning algorithm. In the present world, this is a _centralized model_ as in the researcher must trust the service to run their algorithm according to specification. But if the researcher cannot trust the service, what can we do instead? 

A line of work on PAC verification [GRSY '21] claims a solution by drawing on a long history of interactive proofs. That is, if we allow the researcher and service to communicate, there may be a way for the researcher to verify that the service did in fact do what it was supposed to do. The main question is: how efficient can the verifier be? For example, can the verifier skip over computation, or use less samples? 

In our work, we show how to verify the delegation of a large class of learning algorithms (all statistical query algorithms) where, in some cases, the verifier only needs exponentially fewers data samples than the learning algorithms needs in the first place. Thus, we provide a way for a researcher to check the cloud computing service's work with exponentially less data. We call these _Publicly-Verifiable Certificates of Statistical Validity (pvCSVs)_.

pvCSVs add the additional benefit that they are non-interactive: that is, the services needs to only release a single certificate instead of communicating back and forth with the researcher. Additionally, the certificate is public-verifiable, meaning that any verifier can check if the service's work is statistical valid for each verifier's distribution.

Integral to our exponential separation are interactive fingerprinting codes which show the existence of statistical algorithms that require worst-case sample complexity [SU '15]. Additionally, we adapt the Fiat-Shamir transformation for our delegation setting. The traditional Fiat-Shamir transformation would incurs an blow-up error that is exponential in the number of times a learning algorithm accesses its data. However, we separate the notion of a learning algorithm accessing its data away from when a learning algorithm samples randomness, the number of which we call _epoch complexity_. Therefore, as long as epoch complexity is constant, even an arbitrarily adaptive learning algorithm has a pvCSV with low error!
